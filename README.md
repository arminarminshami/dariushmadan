# داریوش معدن آذر — وب‌سایت شرکتی + پنل مدیریت

سایت شرکتی و پنل مدیریت شرکت **داریوش معدن آذر** (شماره ثبت ۲۰۶۰۴).
ساخته‌شده با **HTML5 + CSS3 + Vanilla JavaScript** خالص — بدون هیچ فریم‌ورک،
بدون بک‌اند، و آماده اجرا با یک بار باز کردن `index.html`.

---

## ۱) اجرا

هیچ نصب یا سروری لازم نیست. کافی‌ست فایل `index.html` را در مرورگر باز کنید.
برای بهترین نتیجه (به‌خصوص برای تست پنل مدیریت با آپلود تصویر)، پیشنهاد می‌شود
با یک سرور محلی ساده اجرا شود، چون بعضی مرورگرها فایل‌های `file://` را محدود می‌کنند:

```bash
# در پوشه اصلی پروژه
python3 -m http.server 8080
# سپس در مرورگر باز کنید:
http://localhost:8080
```

### هاست روی GitHub Pages
این پروژه کاملاً استاتیک است، پس مستقیماً روی GitHub Pages قابل هاست است:
1. محتوای این پوشه را در یک ریپازیتوری گیت‌هاب push کنید.
2. از تنظیمات ریپازیتوری، بخش **Settings → Pages** را باز کرده و شاخه (branch) اصلی را به‌عنوان source انتخاب کنید.
3. سایت روی آدرس `https://<username>.github.io/<repo-name>/` بالا می‌آید.

## ۳) ویرایش محتوا — دو روش

### روش ۱ (پیشنهادی): از طریق پنل مدیریت
آدرس `admin/index.html` را باز کنید.
- **رمز پیش‌فرض ورود: `20604`** (شماره ثبت شرکت) — از بخش «تنظیمات» قابل تغییر است.
- از پنل می‌توانید: پروژه‌ها، اخبار، تصاویر گالری، فایل‌های دانلودی و اطلاعات تماس/شرکت را
  اضافه، ویرایش یا حذف کنید. هر تغییری بلافاصله روی سایت عمومی اعمال می‌شود چون هر دو
  بخش از یک منبع داده مشترک (`DMA_DB` در `assets/js/data.js`) می‌خوانند.

⚠️ **نکته امنیتی مهم:** رمز ورود پنل فقط سمت مرورگر (client-side) بررسی می‌شود و
یک قفل واقعی امنیتی نیست — هرکسی که کد جاوااسکریپت را ببیند می‌تواند از آن عبور کند.
تا زمانی که بک‌اند واقعی اضافه نشده، از این پنل برای داده‌های محرمانه یا حساس استفاده نکنید.

### روش ۲: ویرایش مستقیم کد
داده‌های اولیه (seed) در `assets/js/data.js` داخل شیء `DMA_SEED` قرار دارند.
تغییر این بخش، مقادیر پیش‌فرض را برای بازدیدکنندگانی که هنوز چیزی در localStorage
ذخیره نکرده‌اند عوض می‌کند. متن‌های ثابت هر صفحه هم مستقیم داخل فایل‌های `pages/*.html` است.

## ۴) افزودن پروژه جدید (نمونه ساختار داده)

هر پروژه دقیقاً همین ساختار را دارد (چه از پنل اضافه شود، چه دستی در کد):

```json
{
  "id": 3,
  "name": "نام معدن",
  "location": "استان، شهر",
  "mineral": "نوع ماده معدنی",
  "status": "در حال اکتشاف",
  "description": "توضیح کوتاه...",
  "images": [],
  "documents": [],
  "createdAt": "1404-01-01"
}
```

ساختار مشابه برای خبر (`title, date, content`)، رسانه (`type, url, category`)
و فایل دانلودی (`title, category, fileUrl, date`) هم در `assets/js/data.js` تعریف شده است.

## ۵) نحوه کار پنل مدیریت

- **داشبورد:** آمار کلی (تعداد پروژه، خبر، رسانه، دانلود) + آخرین موارد ثبت‌شده.
- **پروژه‌ها:** افزودن/ویرایش/حذف، آپلود چند تصویر (به‌صورت base64 در همان لحظه پیش‌نمایش داده می‌شود).
- **اخبار:** افزودن/ویرایش/حذف خبر با تاریخ.
- **رسانه:** آپلود گروهی تصویر برای گالری، دسته‌بندی، حذف.
- **دانلودها:** آپلود فایل (PDF یا هر سندی) — فایل به‌صورت data-URL ذخیره می‌شود و دکمه دانلود
  در سایت عمومی واقعاً همان فایل را دانلود می‌کند.
- **تنظیمات:** نام شرکت، شماره ثبت، سرمایه، تلفن، ایمیل، آدرس دفاتر، اینستاگرام، اعضای هیئت‌مدیره،
  و متن‌های «درباره ما» و «چشم‌انداز» — همه این‌ها در هدر/فوتر/صفحه تماس/درباره‌ما به‌صورت خودکار به‌روزرسانی می‌شوند.
- **تغییر رمز پنل:** از پایین صفحه تنظیمات.

⚠️ محدودیت localStorage معمولاً حدود ۵ مگابایت است. تصاویر زیاد یا حجیم می‌توانند این
محدودیت را پر کنند؛ برای استفاده واقعی/تولیدی، تصاویر باید در آینده به یک سرویس
ذخیره‌سازی فایل واقعی (مثل S3 یا سرور خودتان) منتقل شوند.

## ۶) مسیر ارتقا به بک‌اند واقعی (Future Upgrade Path)

طراحی این پروژه از ابتدا برای این آماده شده که جایگزینی localStorage با یک API واقعی
به کمترین تغییر ممکن نیاز داشته باشد:

1. تمام صفحات سایت و پنل مدیریت **فقط** از طریق شیء `DMA_DB` (در `assets/js/data.js`)
   داده می‌خوانند و می‌نویسند — هیچ صفحه‌ای مستقیم به `localStorage` دسترسی ندارد.
2. کافی‌ست بدنه توابع `DMA_DB` (مثل `getProjects`, `saveProject`, `deleteProject`, ...)
   را از خواندن/نوشتن `localStorage` به `fetch()` واقعی به سمت API خودتان تغییر دهید.
   امضای توابع (ورودی/خروجی) همان می‌ماند، پس هیچ صفحه دیگری نیاز به تغییر ندارد.
3. فایل‌های `/data/*.json` دقیقاً ساختار داده‌ای مورد انتظار API را نشان می‌دهند و
   می‌توانند مستقیماً به‌عنوان response نمونه یا seed دیتابیس استفاده شوند.
4. برای آپلود واقعی فایل (به‌جای base64)، کافی‌ست در `admin/js/admin.js` توابع
   `readFileAsDataURL` را با یک `fetch` به سمت اندپوینت آپلود جایگزین کنید و
   آدرس برگشتی فایل را در همان فیلد (`images`, `url`, `fileUrl`) ذخیره کنید.
5. برای امنیت واقعی پنل مدیریت، `admin/js/auth.js` باید با یک سیستم ورود سمت سرور
   (session/JWT) جایگزین شود.

این پروژه عمداً **بدون** بلاک‌چین، هوش مصنوعی، ERP یا سیستم‌های پیچیده سازمانی ساخته
شده — فقط یک پایه تمیز، مقیاس‌پذیر و قابل توسعه.
